## Objective

Implement A* shortest path algorithm - heuristic-guided pathfinding that's faster than Dijkstra when a good heuristic is available. Requires nodes to have coordinates or distance estimates.

## Details

### Type
- [x] Feature - New functionality or enhancement  

### Priority
- [x] P2 - Medium (nice to have)

### Cypher Syntax
```cypher
RETURN astar(source, target, 'lat', 'lon')  -- using lat/lon properties
RETURN astar(source, target, heuristic_func)
```

### Return Format
```json
{"path": ["node1", "node2", "node3"], "distance": 2.5, "nodes_explored": 15}
```

### Complexity
- O(E) best case with perfect heuristic
- O(V log V + E) worst case

### Notes
- Requires heuristic function (Euclidean, Manhattan, Haversine for geo)
- Falls back to Dijkstra if no heuristic available
